# Guía de MDX - Template y Mejores Prácticas

Esta es una guía completa sobre cómo estructurar y escribir archivos MDX en nuestro proyecto. MDX combina la simplicidad de Markdown con el poder de los componentes React.

## 1. Estructura del Frontmatter

El frontmatter debe seguir exactamente esta estructura para que funcione con nuestro sistema de content collections:

```yaml
---
title: "Título del artículo (requerido)"
readtime: 8 # Tiempo estimado de lectura en minutos (requerido)
description: "Descripción breve del contenido" # (requerido)
author: # (opcional)
  name: "Nombre del autor"
  title: "Cargo o posición"
  picture: "URL de la imagen del autor" # (opcional)
  link: "URL del perfil del autor" # (opcional)
---
```

## 2. Formato de Contenido

### Encabezados
Usa los niveles de encabezado apropiados:
- `# H1` - Solo para el título principal
- `## H2` - Para secciones principales
- `### H3` - Para subsecciones
- `#### H4` - Para detalles específicos

### Texto con Formato
- **Texto en negrita** para énfasis importante
- _Texto en cursiva_ para énfasis sutil
- `código inline` para comandos o nombres de archivos
- [Enlaces](https://ejemplo.com) para referencias externas

### Listas
Listas numeradas para pasos secuenciales:
1. Primer paso importante
2. Segundo paso a seguir
3. Tercer paso final

Listas con viñetas para elementos relacionados:
- Punto importante número uno
- Punto importante número dos
- Punto importante número tres

## 4. Componentes React en MDX

Una de las ventajas de MDX es que puedes usar componentes React directamente:

import { Pill } from "@/components/ui/pill";

```jsx
import { Pill } from "@/components/ui/pill";

<Pill variant="blue" size="sm">
  Ejemplo de componente
</Pill>
```

## 5. Mejores Prácticas

### Nomenclatura de Archivos
- Usa nombres descriptivos en minúsculas
- Separa palabras con guiones: `mi-guia-mdx.mdx`
- Evita caracteres especiales y espacios

### Estructura del Contenido
- Comienza con una introducción clara
- Usa encabezados para organizar el contenido
- Incluye ejemplos prácticos cuando sea posible
- Termina con un resumen o conclusión

### SEO y Accesibilidad
- Escribe descripciones claras y concisas
- Usa alt text descriptivo para imágenes
- Estructura el contenido de manera lógica
- Incluye palabras clave relevantes de forma natural

## 7. Checklist de Validación

Antes de publicar tu archivo MDX, verifica:

- [ ] El frontmatter está correctamente formateado
- [ ] Todos los campos requeridos están presentes
- [ ] Los encabezados siguen una jerarquía lógica
- [ ] Los enlaces funcionan correctamente
- [ ] El código está correctamente formateado
- [ ] La descripción es clara y concisa
- [ ] El tiempo de lectura es realista

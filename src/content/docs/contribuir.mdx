# Cómo Contribuir a BuscaRamos

¡Gracias por tu interés en mejorar BuscaRamos! Aceptamos contribuciones tanto de miembros del equipo como de la comunidad externa. Aquí te explicamos cómo puedes participar.

---

## 1. Para miembros internos del equipo

- Crea una rama descriptiva para tu cambio (por ejemplo, `userName/featureName` o `userName/bug-fix`).
- Sigue las convenciones de código y asegúrate de que tu cambio pase los tests y linting.
- Abre un Pull Request (PR) hacia la rama development para revisión.

---

## 2. Para colaboradores externos (comunidad)

¡Eres bienvenido a contribuir! Sigue estos pasos:

1. **Haz un fork** del repositorio en tu cuenta de GitHub.
2. Crea una rama descriptiva para tu cambio (por ejemplo, `fix/bug-busqueda` o `feature/nueva-seccion-faq`).
3. Realiza tus cambios y asegúrate de que el proyecto siga funcionando localmente:
   - Instala dependencias (`pnpm install`)
   - Ejecuta el servidor en modo preview (`pnpm preview`)
   - Verifica que no haya errores en la consola
4. Haz commit y push a tu fork.
5. Abre un Pull Request (PR) hacia la rama development del repositorio original.
6. Describe claramente tu aporte y, si es posible, adjunta capturas o ejemplos.

El equipo revisará tu PR y te dará feedback o lo integrará si todo está correcto.

---

## 3. Buenas prácticas

- Escribe código claro y documentado.
- Prefiere los estilos y componentes existentes.
- Si tu cambio afecta la UI, verifica que sea responsive y accesible.
- Consulta la documentación interna y el README para detalles técnicos.

---

¿Tienes dudas? Abre un issue o contacta al equipo de Osuc.
